# 画面設計

## 画面一覧

| 画面 | パス | 説明 |
|------|------|------|
| トップ / LP | `/` | サービス説明・診断スタートボタン |
| 診断画面 | `/diagnosis` | 質問ステップ（SPA） |

## 1. トップページ（`/`）

### 目的
「このサービスが何をするものか」を5秒で理解させ、診断を開始させる。

### レイアウト構成

```
┌─────────────────────────────┐
│  [ロゴ / サービス名]          │ ← ヘッダー
├─────────────────────────────┤
│                             │
│  【キャッチコピー】           │ ← Hero セクション
│  副業を当てるより、           │
│  あなた自身を知ることから。   │
│                             │
│  [副コピー テキスト]         │
│  自分でも気づいていない強みを │
│  AIが言語化します。           │
│  10問・5分で、今日の一歩へ。  │
│                             │
│  ┌───────────────────────┐  │
│  │  無料で診断をはじめる  │  │ ← CTAボタン（大・目立つ色）
│  └───────────────────────┘  │
│                             │
├─────────────────────────────┤
│  ▼ こんな方におすすめ        │ ← 共感セクション
│  ✓ 副業したいけど何から？     │
│  ✓ 自分に向いている副業は？  │
│  ✓ 営業なしで稼ぎたい        │
│  ✓ まず5万円から始めたい     │
├─────────────────────────────┤
│  ▼ 3ステップで結果が出る     │ ← 説明セクション
│  ① 12問に答える（5分）       │
│  ② あなたの強みを言語化      │
│  ③ 向いている副業と一歩を提示 │
├─────────────────────────────┤
│  ▼ 対象副業（18種）          │ ← 信頼感セクション
│  スキル系・コンテンツ系の     │
│  副業一覧をタグ表示           │
├─────────────────────────────┤
│  ┌───────────────────────┐  │
│  │  無料で診断をはじめる  │  │ ← 2つ目のCTAボタン
│  └───────────────────────┘  │
├─────────────────────────────┤
│  登録不要・無料・5分で完了    │ ← フッター（信頼ライン）
└─────────────────────────────┘
```

### UX方針
- ファーストビューにCTAボタンを配置
- 抽象的な説明より「何が得られるか」を具体的に
- 「登録不要・無料・5分」を複数箇所で訴求

## 2. 診断画面（`/diagnosis`）

### 目的
離脱させずに12問を完了させる。ストレスフリーな操作性。

### レイアウト構成

```
┌─────────────────────────────┐
│  副業コンパスAI              │ ← ロゴ（小）
├─────────────────────────────┤
│  ████████░░░░░░░░░  4 / 12  │ ← プログレスバー
├─────────────────────────────┤
│                             │
│  Q4.                        │ ← 質問番号
│  仕事でよく任される役割は？  │ ← 質問文（大きめ）
│                             │
│  ┌─────────────────────┐    │
│  │ A 資料作成・レポート  │    │ ← 選択肢カード
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ B チェック・確認作業  │    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ C 新ツールの検証・導入│    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ D 後輩・部下の指導   │    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ E 企画・アイデア出し  │    │
│  └─────────────────────┘    │
│                             │
│  ← 前の質問に戻る            │ ← 戻るリンク（小）
└─────────────────────────────┘
```

### インタラクション
- 選択肢をタップ/クリックしたら即座に次の質問へ（「次へ」ボタン不要）
- 選択した瞬間にハイライト → 0.3秒後に次の質問へ遷移（アニメーション）
- 最終問（Q12）だけ「結果を見る」ボタンを表示

### UX方針
- 選択肢はテキストのみ・シンプルなカード形式
- 1画面1質問・迷わせない
- スマホでも親指だけで操作できるサイズ
- 「前に戻る」は目立たせすぎない（離脱防止）

## 3. 結果画面（`/result`）

### 目的
単なる診断結果ではなく「自分への気づき」を体験させ、行動意欲を引き出す。

### レイアウト構成

```
┌─────────────────────────────┐
│  診断が完了しました！         │ ← ヘッダー
├─────────────────────────────┤
│                             │
│  ── Section 1 ──            │
│  あなたの行動特性            │ ← タグカード形式
│                             │
│  [細かい作業が得意] [調べる] │
│  [コツコツ継続]   [整理整頓] │
│                             │
├─────────────────────────────┤
│  ── Section 2 ──            │
│  無意識にできていること       │ ← 2〜3文の自然な文章
│                             │
│  あなたは情報を整理して体系   │
│  的に伝えることが得意です。   │
│  相手が見落としがちな細部を   │
│  自然に気にかける力があります。│
│                             │
├─────────────────────────────┤
│  ── Section 3 ──            │
│  あなたの働き方タイプ        │
│                             │
│  ┌─────────────────────┐    │
│  │  📋 受託スキル型      │    │ ← タイプカード
│  │  確かなスキルで        │    │
│  │  着実に稼ぐ人          │    │
│  │  [詳しい説明テキスト]  │    │
│  └─────────────────────┘    │
│                             │
│  向いていない働き方:          │
│  資産構築型（理由）          │
│                             │
├─────────────────────────────┤
│  ── Section 4 ──            │
│  おすすめ副業ランキング       │
│                             │
│  🥇 1位 Webライティング      │
│  [マッチ理由のテキスト]      │
│  月収目安: 3〜5万円（半年後）│
│                             │
│  🥈 2位 QAテスター          │
│  [マッチ理由のテキスト]      │
│                             │
│  🥉 3位 テンプレート販売     │
│  [マッチ理由のテキスト]      │
│                             │
├─────────────────────────────┤
│  ── Section 5 ──            │
│  おすすめしない副業          │
│                             │
│  ✗ アフィリエイト            │
│  [向いていない理由]          │
│                             │
├─────────────────────────────┤
│  ── Section 6 ──            │ ← 強調表示（色付きカード）
│  🎯 今日やること             │
│                             │

│  Webライティングの案件を3つ  │

│  （所要時間：10分）          │
│                             │
├─────────────────────────────┤
│  ── Section 7 ──            │
│  7日間ロードマップ           │
│                             │
│  Day 1: 今日やることを実行   │
│  Day 2: クラウドワークスで   │
│         ポートフォリオ確認   │
│  Day 3: 初案件に応募する     │
│  Day 4: 作業環境を整える     │
│  Day 5: 1本書いてみる       │
│  Day 6: フィードバックをもらう│
│  Day 7: 次の案件を探す       │
│                             │
├─────────────────────────────┤
│  [Xでシェアする]             │ ← シェアボタン
│  [もう一度診断する]          │ ← やり直しボタン
│  [この結果は役に立ちましたか？]│ ← フィードバック
└─────────────────────────────┘
```

### UX方針
- Section 1〜2（自己認識）を先に見せる。副業名より「気づき」を先行させる
- Section 6（今日やること）は視覚的に目立たせる（色付きカード・アイコン）
- 7日間ロードマップはシンプルなリスト形式（チェックボックスは将来機能）
- ネガティブな表現を使わない（「苦手」ではなく「向いていない理由」として説明）
- 結果のシェアはXのみ（MVP）

## コンポーネント設計方針

```
app/
├── page.tsx              ← トップページ
├── diagnosis/
│   └── page.tsx          ← 診断画面
└── result/
    └── page.tsx          ← 結果画面

components/
├── diagnosis/
│   ├── QuestionCard.tsx  ← 質問表示
│   ├── OptionCard.tsx    ← 選択肢カード
│   └── ProgressBar.tsx   ← プログレスバー
└── result/
    ├── TraitTags.tsx     ← 行動特性タグ
    ├── StrengthText.tsx  ← 強みの言語化
    ├── WorkStyleCard.tsx ← 働き方タイプ
    ├── SideHustleRank.tsx← 副業ランキング
    ├── TodayAction.tsx   ← 今日やること
    └── Roadmap.tsx       ← 7日間ロードマップ
```

### 疎結合設計の方針
- 結果データはJSON形式のオブジェクトで管理（後からDB保存・PDF出力に対応しやすくする）
- ロジック（`lib/`）とUIコンポーネント（`components/`）を完全に分離する
- 質問データ・副業データは`data/`フォルダのJSONで管理（ロジックと分離）
