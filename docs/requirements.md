# 要件定義

## 1. サービス目的

副業を診断するのではなく、ユーザー本人も気づいていない行動特性・強み・行動パターンを言語化し、
向いている副業と最初の一歩を提示するサービス。

「自分には何もない」と思っている人が「実はこんな特徴があった」と気づき、
その日のうちに最初の一歩を踏み出せることを最終目的とする。

---

## 2. ターゲットユーザー

### プライマリ
- 年齢: 30〜50代
- 属性: 会社員（正社員・契約社員）
- 課題: 副業を始めたいが何から始めていいかわからない
- 特性: AI初心者・営業が苦手・在宅でできることをやりたい

### セカンダリ
- 主婦・主夫（スキマ時間で稼ぎたい）
- フリーランス志望（独立前の副業から始めたい）

---

## 3. MVPの機能要件

### 3-1. 診断機能

| 要件 | 詳細 |
|------|------|
| 質問数 | 12問（10〜15問の範囲） |
| 質問形式 | 選択式（カード形式）、自由入力は最小限 |
| 回答時間 | 5分以内 |
| 質問内容 | 抽象的な自己評価ではなく、過去の事実・行動に基づく |
| 進行 | 1問ずつ表示・プログレスバーあり |
| 戻る機能 | 前の質問に戻って回答を変更できる |
| 診断の単位 | SPAで画面遷移なく進む |

### 3-2. 結果表示機能

以下を全て無料・登録なしで表示する。

| セクション | 内容 |
|-----------|------|
| 行動特性 | ユーザーの行動パターン・特徴（3〜5項目・タグカード形式） |
| 強みの言語化 | 無意識にできていること（2〜3文の自然な文章） |
| 向いている働き方タイプ | 5タイプから最適タイプを提示＋詳細説明 |
| 向いていない働き方タイプ | 理由付きで提示（ネガティブにならない表現） |
| おすすめ副業ランキング | 上位3〜5件・マッチ理由付き |
| おすすめしない副業 | 2〜3件・理由付き（攻めすぎない表現） |
| 今日やること | 具体的なアクション1つ（5分でできるレベル） |
| 7日間ロードマップ | 1日ごとの行動計画（1行で具体的に） |

### 3-3. その他機能

| 機能 | 要件 |
|------|------|
| やり直し | 診断をリセットして最初からやり直せる |
| シェア | 結果をX（Twitter）でシェアできる |
| フィードバック | 結果への満足度を収集できる（Google Forms埋め込み等） |

---

## 4. 非機能要件

| 要件 | 詳細 |
|------|------|
| 認証 | 不要（MVP） |
| 料金 | 全機能無料（MVP） |
| モバイル対応 | スマートフォンで操作できること（レスポンシブ） |
| 表示速度 | トップページFCP 2秒以内 |
| データ保存 | MVPはセッション内メモリのみ・サーバー保存なし |
| セキュリティ | 個人情報・ログイン不要のため最小限 |

## 5. 対象副業の定義

### 対象（MVP：18種）

#### スキル系（8種）
| 副業名 | 特徴 |
|--------|------|
| Webライティング | 文章力を活かす・初心者でも始めやすい |
| QAテスター | 確認・品質管理スキルを活かす |
| データ入力 | PCスキルがあれば始めやすい |
| LP制作 | ページ1枚の制作 |
| ホームページ制作 | Webサイト全体の制作 |
| ノーコード制作 | プログラミング不要でサイト・アプリ制作 |
| AI活用代行 | AIツールを使って業務を代行 |
| MEO運用代行 | Googleマップの最適化支援 |

#### コンテンツ系（10種）
| 副業名 | 特徴 |
|--------|------|
| note販売 | 知識・体験を記事・有料マガジンで販売 |
| BOOTH販売 | デジタルコンテンツ・素材を販売 |
| AIプロンプト販売 | AI活用のプロンプトを販売 |
| テンプレート販売 | 仕事で使えるテンプレートを販売 |
| YouTube BGM | 著作権フリー音楽を配信 |
| YouTube解説 | 知識・スキルを動画で解説 |
| ブログ | 記事を積み上げ広告収入を得る |
| アフィリエイト | 商品紹介で成果報酬を得る |
| AIツール販売 | AIを使ったツール・サービスを販売 |
| 業務自動化ツール制作 | 業務効率化ツールを制作・販売 |

### 対象外（MVP）
せどり / 転売 / 投資 / FX / 暗号資産 / ギャンブル / 高額スクール前提 / 資格必須ビジネス

## 6. 将来拡張の前提

MVPの設計は以下の追加を見越して疎結合に実装する。

| 将来機能 | 設計上の前提 |
|---------|------------|
| Claude API連携 | タグ・プロファイルをJSON形式でAIに渡せる構造にする |
| 詳細診断 | 質問セットを差し替え・追加できる構造にする |
| ユーザー認証 | 認証レイヤーを後付けできる疎結合設計にする |
| DB保存 | 回答・プロファイルをJSON形式で保存できるデータ構造にする |
| PDF出力 | 結果コンポーネントを独立させて後からPDF化できるようにする |

## 7. MVPの検証ゴール

| 指標 | 目標値 | 計測方法 |
|------|--------|---------|
| 診断完了率 | 70%以上 | Vercel Analytics |
| 結果への納得感 | 「参考になった」70%以上 | フィードバックフォーム |
| 行動意欲 | 「今日やってみたい」60%以上 | フィードバックフォーム |
